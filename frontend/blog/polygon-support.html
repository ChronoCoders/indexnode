Polygon Support Now Live

IndexNode now indexes Polygon (PoS) with the same Merkle proofs, on-chain timestamping, and IPFS storage as Ethereum. Same trust model, significantly lower indexing costs.

Distributed Systems Labs

Engineering Team

What's Available Today

Polygon (PoS) is now a first-class supported chain in IndexNode. You can create indexing jobs for any Polygon contract, receive events via GraphQL and WebSocket subscriptions, and verify data integrity via Merkle proofs — exactly as you would for Ethereum.

To target Polygon, set chain: "polygon" in your indexing job: 

mutation {
  createBlockchainJob(input: {
    contractAddress: "0x7ceB23fD6bC0adD59E62ac25578270cFf1b9f619",
    chain: "polygon",
    events: ["Transfer"],
    fromBlock: 55000000
  }) {
    id
    status
  }
}

The dashboard's "Create Indexing Job" form also lists Polygon in the chain selector alongside Ethereum.

Why Polygon Matters for Web3 Infrastructure

Polygon processes blocks roughly every two seconds and its gas costs are a fraction of Ethereum mainnet. For applications that need high-throughput event monitoring — gaming platforms, NFT marketplaces, micro-payment rails — Ethereum's block times and gas costs are prohibitive. Polygon handles these workloads efficiently while maintaining EVM compatibility.

For IndexNode users, this means you can index high-frequency contracts without consuming excessive credits — Polygon's faster block cadence and lower gas costs translate to lower on-chain commitment costs when we anchor Merkle roots.

The Same Verification Guarantees

Polygon support is not a cut-down version of our Ethereum integration. Every feature available on Ethereum is available on Polygon:

  • Merkle proof generation and verification for every event batch
  • On-chain Merkle root commitment via the TimestampRegistry contract deployed on Polygon
  • IPFS backup for all indexed event batches (Growth and Enterprise)
  • Real-time WebSocket subscriptions for live event streams
  • AI extraction for structured data from event parameters

Verification works identically regardless of chain. Pass the contentHash of any event to verifyHash and the API will return the block number and transaction hash of the on-chain commitment — whether it was anchored on Ethereum or Polygon.

Existing Polygon Users

If you've been waiting to migrate from another indexer for Polygon contracts, now is the time. Historical backfill is supported — set fromBlock to the deployment block of your contract to build a complete event history from genesis.

Credit costs for Polygon jobs are consistent with Ethereum at the same per-event rate.

What's Next

We're evaluating additional EVM-compatible chains based on user demand. If you need support for a specific network, reach out at hello@indexnode.io — network additions are prioritized by active user requests.
