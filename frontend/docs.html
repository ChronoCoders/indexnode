Documentation

Welcome to IndexNode. This guide will help you integrate blockchain event indexing with cryptographic verification into your application.

API Endpoint

https://api.indexnode.io/v1/graphql

Quick Start

Get started with IndexNode in 5 minutes.

1. Get Your API Key

Sign up and generate an API key from your dashboard.

curl https://api.indexnode.io/v1/auth/signup \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "password": "your_password"}'

2. Make Your First Query

Query blockchain events using GraphQL.

query GetEvents {
  blockchainEvents(limit: 10) {
    id
    contractAddress
    eventName
    blockNumber
    transactionHash
  }
}

3. Verify Data

Verify event data with on-chain Merkle proofs.

query VerifyEvent {
  verifyMerkleProof(
    eventId: "evt_123"
  ) {
    isValid
    blockNumber
    merkleRoot
    proof
  }
}

Queries

blockchainEvents

Retrieve blockchain events with filtering and pagination.

query BlockchainEvents {
  blockchainEvents(
    contractAddress: "0x..."

    eventData
    contentHash
    ipfsCid
  }
}

Verify an Event On-Chain

IndexNode does not expose a separate verification query. Instead, use the contentHash returned by blockchainEvents to independently verify the event against the TimestampRegistry contract on-chain. No dependency on IndexNode required.

# 1. Fetch the event's content hash
query GetEvent {
  blockchainEvents(
    contractAddress: "0x..."
    limit: 1
  ) {
    id
    contentHash
    transactionHash
  }
}

# 2. Check the hash against TimestampRegistry on-chain
# Call: TimestampRegistry.getCommitment(contentHash)
# Returns the block number it was committed in — verifiable by anyone

Retrieve IPFS Content

Use the ipfsCid from any event to fetch content directly from IPFS — no API call required. Content is pinned via Pinata and available on any public gateway.

# Fetch via public gateway (no auth required)
https://gateway.pinata.cloud/ipfs/{ipfsCid}

# Or any other IPFS gateway
https://ipfs.io/ipfs/{ipfsCid}

On-Chain Timestamping

Create immutable, cryptographic proof of when your data was indexed.

How It Works

  1. 1. IndexNode indexes blockchain events and computes a SHA-256 hash of the event data
  2. 2. Content hashes are batched into a Merkle tree for efficient verification
  3. 3. The Merkle root is committed to an Ethereum or Polygon smart contract
  4. 4. You receive a transaction hash and block number as proof of timestamp

Query Timestamp Proof

query GetEvent {
  blockchainEvents(
    contractAddress: "0x..."
    limit: 1
  ) {
    contentHash
    ipfsCid
    blockNumber
    transactionHash
  }
}

IPFS Storage

Store indexed blockchain data on IPFS with automatic pinning via Pinata.

Note: All indexed events are automatically backed up to IPFS on Pro and Enterprise plans. Free tier requires manual upload.

Merkle Verification

Verify event authenticity using cryptographic Merkle proofs.

How Merkle Proofs Work

Merkle trees allow efficient verification of large datasets. Instead of checking every event on-chain, you only need:

  • The event's content hash
  • A small proof (sibling hashes)
  • The Merkle root (stored on-chain)

Verify an Event

# Fetch contentHash from the API
query GetEvent {
  blockchainEvents(
    contractAddress: "0x..."
    limit: 1
  ) {
    id
    contentHash
  }
}

# Verify on-chain: TimestampRegistry.getCommitment(contentHash)
# Returns: block number the hash was committed in
# Independently verifiable on any Ethereum/Polygon RPC

Manual Verification

You can verify proofs client-side using standard Merkle tree libraries:

import { MerkleTree } from 'merkletreejs';
import { keccak256 } from 'ethereum-cryptography/keccak';

const leaf = keccak256(contentHash);
const verified = MerkleTree.verify(
  proof,
  leaf,
  merkleRoot
);

console.log('Verified:', verified);

HTTP Examples (Python)

The GraphQL API works directly from any HTTP client. Below are ready-to-use fetch examples for Node.js and browsers.

Installation

pip install indexnode

Usage

from indexnode import IndexNodeClient

# Initialize client
client = IndexNodeClient(api_key="your_api_key")

# Query events
events = client.query("""
    query {
        blockchainEvents(limit: 10) {
            id
            contractAddress
            eventName
            blockNumber
        }
    }
""")

for event in events['blockchainEvents']:
    print(f"Block: {event['blockNumber']}")

# Verify event
verification = client.verify_event(event_id="evt_abc123")
if verification['isValid']:
    print("Event verified on-chain!")

# Upload to IPFS
result = client.upload_to_ipfs(content="event_data", pin=True)
print(f"CID: {result['cid']}")

Rust SDK

High-performance Rust client for IndexNode API.

Installation

# Cargo.toml
[dependencies]
indexnode = "0.1"
tokio = { version = "1", features = ["full"] }

Usage

use indexnode::IndexNodeClient;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Initialize client
    let client = IndexNodeClient::new("your_api_key");
    
    // Query events
    let events = client.query_events(
        QueryParams {
            contract_address: Some("0x...".to_string()),
            limit: Some(10),
            ..Default::default()
        }
    ).await?;
    
    for event in events {
        println!("Block: {}", event.block_number);
    }
    
    // Verify event
    let proof = client.verify_event("evt_abc123").await?;
    if proof.is_valid {
        println!("Event verified on-chain!");
    }
    
    Ok(())
}

Best Practices

Use Cursor-Based Pagination

For large datasets, use cursor-based pagination instead of offset to avoid missing records.

Cache Merkle Proofs

Store Merkle proofs locally to avoid repeated verification queries.

Use WebSocket for Real-Time Data

Subscribe to events via WebSocket instead of polling for better performance.

Rate Limiting

Implement exponential backoff when hitting rate limits (429 responses).

Frequently Asked Questions

What blockchains are supported?

Ethereum and Polygon.

How long does verification take?

On-chain verification typically takes 2-5 minutes after event indexing, depending on blockchain confirmation times.

Can I index historical events?

Yes! Specify a fromBlock when creating an indexing job to backfill historical data from any block.

What are rate limits?

Free tier: 100 requests/hour. Pro: 10,000 requests/hour. Enterprise: Custom limits. Rate limits reset hourly.
